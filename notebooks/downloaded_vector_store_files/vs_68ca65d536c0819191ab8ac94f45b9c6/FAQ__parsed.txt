      "text": "---\ntitle: BHTOM FAQ (GUI & API)\ntags: [faq, gui, api, uploads, targets, observatories]\nupdated: 2025-09-17\nversion: bhtom2\n---\n\n> Quick answers to common questions about using the BHTOM web app (GUI) and API.\n\n## General\n\n**Q: What is BHTOM?**  \nA: A web platform for time-domain astronomy: create/track targets, upload photometry/spectra, and share results with your team.\n\n**Q: Do I need an account?**  \nA: Yes. Registration is human-approved. After sign-up, fill *About me* and wait for a confirmation email.\n\n**Q: Where do I find help?**  \nA: Use the chat bubble (bottom-right) and search this FAQ. For advanced topics, see API docs in About section.\n\n---\n\n## Accounts & Access\n\n**Q: How do I get my API token?**  \nA: Go to **your_name (profile) → Copy Token (button) ** pressing the button, copies it to your clipboard. Treat it like a password.\n\n**Q: I can’t see some targets or data.**  \nA: Visibility may be restricted by group/permissions. Ask an admin to add you to the relevant group.\n\n---\n\n## Targets\n\n**Q: Why I can not find my target with Importance=0**\nA: By default, in the target list, we set the filtering to Importance>0, to display and list only targets requested for observation. You can modify this filter by scrolling down below the target list and find the Filtering options. Remove 1 in Importance and set to 0 or leave blank.\n\n**Q: How do I create a target in the GUI?**  \nA: **Targets → Create**. Fill **Name**, **RA**, **Dec**, optional **Classification**, **Description**, **Importance (0–10)**, and **Cadence (days)**. Click **Save**.\n\n**Q: Accepted coordinate formats?**  \nA: RA/Dec in sexagesimal (`hh:mm:ss.s`, `±dd:mm:ss.s`) or decimal degrees. The form validates on submit.\n\n**Q: Can I import many targets at once?**  \nA: Yes. Use **Targets → Import** with a CSV. **Headers are case-sensitive**. Start with a small test file. You can upload 500 targets in one go.\n\n**Q: Where do I find a target after creation?**  \nA: **Targets → List**, search by name; or go directly to `/targets/<Name>/`.\n\n**Q: Why the target's light curve is empty after creation?**\nA: Once the target is created, the harvesters are run in the background. It can take couple of minutes for our harvesters to find the data for your target. They query time-domain archives, for example, ZTF, ASASSN, Atlas, OGLE, Gaia, CRTS, LINEAR, NEOWISE (infrared), LOFAR (radio), SDSS, PS1, DECAPS, 2MASS (infrared). In case of Atlas, it might take up to couple of hours for the data to appear on the light curve.\n---\n\n## Uploads (Photometry / FITS / SExtractor)\n\n**Q: How do I upload FITS files in the GUI?**  \nA: Open the target page, find tab → **Manage Data**. Select up to **5 files** per batch (for large sets, use the API). Choose **Observatory** and **Camera**. By default, the logged-in person is assumed to be the observer, you can edit this and add/remove other people, but they have to be registered in BHTOM. Note, the observation (we call it Reduced Datum) will still be tagged with the user whose token was used to upload that given datapoint (we call it Data Product).\n\n**Q: How do I upload SExtractor photometry?**  \nA: **Target** -> **Manage Data**. Check Photometry in SExtractor format, select your files, select the observatory/camera. In contrast to FITS upload, here you also need to specify the MJD of the observation. \n\n**Q: What is the Force Filter field in uploading data products?**\nA: You can leave it at the default (GaiaSP/any). This means that your photometry (FITS or SExtractor) will be matched to all possible Gaia Synthetic Photometry filters (ugriz and UBVRI) and the best match will be used to standardise your photometry. However, if you know that your data is always taken in Sloan filters, you can select GaiaSP/ugriz. This helps avoid some errors in calibration - sometimes for example r and R filters might look alike. The same applies to API upload. You can also narrow down the filter to exact filter (e.g. GaiaSP/r), but it is not recommended (you can be surprised how often the images are mislabelled with wrong filters! We can figure out your filter!). \n\n**Q: I'm observing with RGB camera, which filter set should I use in Force Filter?**\nA: As usual, the default GaiaSP/any, should do the job - it will figure out the standard filter best matching your data. From our experience, however, we found that GaiaSP/R works well for the R filter and GaiaSP/V works for G. With B it might get tricky, so you can first try with GaiaSP/UBVRI to check.\n\n**Q: What is the Dry Run option in upload/Manage Data**\nA: If you tick that box, your data will not get stored in the database and will not appear on the light curve. This is useful for testing how does your data process. \n\n**Q: My upload is “pending” / not visible.**  \nA: Sometimes the server can get overloaded and there is a long queue of files to be processed. Be patient. If your data does not appear within 24h, get in touch.\n\n**Q: What is the best way of contacting the BHTOM team?**\nA: We have an email address contact@bhtom.space. We also have a dedicated Slack Workspace, you can join it via slack.bhtom.space link.\n\n**Q: Pre-processing requirements for FITS?**  \nA: Bias/dark/flat corrected is expected. No need for Astrometric/WCS solution, we perform it anyway. \n\n---\n\n## Observatories & Cameras\n\n**Q: How do I add my observatory/camera?**  \nA: **Observatory** tab first shows your Favourite Observatories list. This is a list of registered observatories you can upload the data for. You can add observatories to this list by clicking the Observatories tab next to Favourite Observatory. It contains the full list of all observatories registered in BHTOM. Then you click the red button Add new observatory. In the new form, you can select an existing observatory from the list. New cameras require admin approval and will get an **ONAME** - a unique name which should be used in any API.\n\n**Q: I can’t find my observatory in uploads.**  \nA: It must be **registered and active** and also present in your Favourite Observatory list. \n\n---\n\n## Data & Visualization\n\n**Q: Where do I see recent photometry for a target?**  \nA: On the target page under **Photometry**. Interactive plots and **Download** are available.\n\n**Q: Can I download my data?**  \nA: Yes. Use **Download** on the target page or the API endpoints for programmatic access.\n\n**Q: What is “Importance” and “Cadence”?**  \nA: *Importance* (0–10) hints at priority; *Cadence* is desired time between observations (days). These values can be updated while the target evolves. If Importance is set to 0, it means it does not require any additional follow-up observations. Check for any additional information on how to observe a given target in the Description as well as in the Comments. The first comment in the target reveals who created the target, hence this would be the best person to ask for any more details regarding the observations strategy. \n\n---\n\n## API\n\n**Q: Where are the API docs?**  \nA: In BHTOM/About tab, there is a link to the Documentation (on Github), follow this and look for API documentation link there. \n\n**Q: How do I authenticate?**  \nA: For API use, you will need your token - it is a long key in form of a string. You can find it in BHTOM, when you click on your name and in your profile page find the Copy Token - this copies the token to your clipboard, so you can then paste it into your code."
