      "text": "---\ntitle: Create a Target (GUI)\ntags: [gui, targets, create, add]\nupdated: 2025-09-17\nversion: bhtom2\n---\n\n## Overview\nTargets in BHTOM correspond to observing requests. Every target created in BHTOM is assumed it requires more or less additional follow-up observations with the telescope network. The only exception is when the Importance of the target is set to 0, it means the target is no longer requested for observation. The importance can change during the evolution of the target, for example, if a supernova is young, it typically has high importance, but when it fades and is no longer visible, we modify its importance to 0. Targets created with Importance=0 can also be used solely for investigating any archival time-domain data, which BHTOM is providing. \nThere are three ways of creating a target in BHTOM. \n- manual form\n- import of a longer list\n- by searching in various catalogues of targets\n\n## Prerequisites\n- Logged in; required permissions (if any)\n- No observatory needed\n\n## Steps - manual form\n1) The main (top) menu, **Targets → Create**.\n2) Fill **Name** (unique, avoid spaces and special characters), **RA**, **Dec**, **Classification** (if known), **Description** (required, can contain any details on the target and the requested way of observing it), **Importance (0–10)** (if 0, the target will not be observed), **Cadence (days)**, **Discovery Date** (can be left empty).\n3) There is no need to fill the Aliases - BHTOM will query all possible catalogues and will append the Aliases if target is found in other surveys, e.g. ASASSN, OGLE, Gaia, TNS\n3) Click **Save** → you land on the target page.\n4) If a target already exists in BHTOM, or a target with a different name, but with coordinates within 3 arcsec radius if found, such target will not be created - you will get an error and the name (with link) to the existing target will apear below the name field. If your target exists under a different name, which is not listed in BHTOM, you can add it as an Alias, or update the target's description, or add comment. \n\n## Steps - import\n1) The main (top) menu, **Targets  → Import**\n2) Upload a .csv to import targets in bulk.\n3) Optional: If you want to add all imported targets to a new group, please fill in the \"Group name\" field.\n\n\n4) Allowed field names in the csv file you want to import:\nname, ra, dec, epoch, parallax, pm_ra, pm_dec, distance, distance_err,\nclassification, description, discovery_date, importance, cadence, phot_class,\ndescription, epoch_of_elements, mean_anomaly, arg_of_perihelion, eccentricity,\nlng_asc_node, inclination, mean_daily_motion, semimajor_axis, epoch_of_perihelion,\nephemeris_period, ephemeris_period_err, ephemeris_epoch, ephemeris_epoch_err, perihdist\n\n5) List of available classifications:\n Be-star outburst, AGN, BL Lac, CV, CEPH, EB, Galaxy, LPV, LBV, M-dwarf flare,\n Microlensing Event, Nova, Peculiar Supernova, QSO, RCrB, RR Lyrae Variable,\n SSO, Star, SN, Supernova imposter, Symbiotic star, TDE, Variable star-other,\n XRB, YSO\n\n6) CSV file format examples:\nname,       type,       ra,         dec,        redshift,   distance,   classification, description\nmytarget,   SIDERAL,    123.12,     -12.34,     2.35,       1.0         Star            nice supernova\nname,       ra,        dec,     importance, cadence\nmytarget,   123.12,    -12.34,  5,          1\nname,       GAIA_ALERTS_name\nmytarget,   Gaia20dup\nname,       GAIA_ALERTS_name,   cadence\nmytarget,   Gaia20dup,          3\n\n7) In these special cases, the Gaia Alerts harvester will gather all information from Gaia Alerts, but any extra columns in the CSV file with corresponding fields will replace those read from Gaia Alerts.\n8) You can upload max 500 targets.\n9) Hit **Upload**\n10) If a target from your list already exists in BHTOM, or a target with a different name, but with coordinates within 3 arcsec radius if found, such target will not be imported and you will get a warning message. \n\n## Steps - Catalog search\n1) Top menu, **Targets  → Catalog search**\n2) Provide the name of the target and then choose from the dropdown list the survey you want to query.\n3) Currently the following catalogs/surveys are available for querying:\n- Gaia Alerts\n- Transient Name Server (TNS)\n- ANTARES Broker\n- OGLE EWS (Early Warning System) - microlensing events\n- NED\n- Simbad\n- Gaia DR3 (sourceid)\n- OGLE OCVS - variable stars from OGLE-IV and OGLE-III\n4) If the target is found, the create target form is pre-filled with details extracted from the survey. Verify them, you can change the Importance if needed and edit the Description field.\n5) Hit **Save** and target is created\n\n## Field reference\n| Field | Required | Format / Choices | Notes |\n|------|----------|------------------|-------|\n| Name | yes | text | Must be unique |\n| RA   | yes | hh:mm:ss.s or decimal | Validated on submit |\n| Dec  | yes | ±dd:mm:ss.s or decimal | — |\n| Epoch | yes | 2000 | epoch of the coordinates |\n| Classification | no | dropdown | Default is Unknown |\n| Description | yes | Text | - |\n| Discovery Date | no | YYYY-MM-DDTHH:MM:SS | can be left blank |\n| Importance | yes | 0–10 | if =0, target will not be requested for observation |\n| Cadence | yes | integer days | — |\n\n## Common errors & fixes\n- **special characters in the name** - avoid\n- **RA/Dec parse error** → check separators / sign.\n- **Description too short** - minimum 10 characters\n\n## See also\n- [Upload FITS (GUI)](./UploadFITS.md), [Import Targets (CSV)](./ImportTargets.md)\n\n## Questions this page answers\n- “How do I create a target?”\n- “What fields are required to add a target?”\n- \"How to add a target to BHTOM which was found by other survey?\"\n- \"Can I create many targets at once?\"\n"
