      "text": "# Target Detail Page\n\nThis document describes the **Target** page (individual target details). The page can be opened:\n- from any **Target List** (Table / Aladin / Visual List), by clicking a target name,\n- directly via **Target ID**,\n- directly via **Target Name** (but only using the main name of the target, not its alias), e.g. https://bh-tom2.astrouw.edu.pl/targets/Gaia20fnr/\n\n---\n\n## Page layout\n\nThe page is split into two columns:\n\n- **Left column:** compact **Target Summary** – sky position and other details on the target.\n- **Right column:** a **tabbed workspace** with in-depth data (Photometry, Models, Spectroscopy, Publication, Manage Data, Manage Groups).  \n  Comments are shown under the tabs (threaded discussion for the target).\n\n---\n\n## Left column — Target Summary\n\n### Name\nThe primary display name of the target - its official name within BHTOM\n\n### Ra,Dec\nEquatorial Ra and Dec are provided as both decimal degrees and hexadecimal numbers (hours, minutes, seconds for Ra and degrees, minutes, seconds for Dec). They are listed next to another for easy copy-pasting.\n\n### Galactic (l,b)\nGalactic coordinates in decimal degrees\n\n### Constellation\nConstellation where the target is located. Latin name of the constellation for a given Ra,Dec.\n\n### Discovered\nThe date and time (UT) when the target was discovered (if available).\n\n### Class (Classification)\n- Scientific type, e.g., **CV**, **SN**, **QSO**, **Variable star**.\n- It is only set when the type of the target is confirmed. If this is a candidate, leave it as unknown and write the candidatae type into the Description field.\n- Default is **Unknown**\n\n### Description\nHuman-readble longer text. Should contain any information about the target relevant for its identification and the follow-up. It can specify that the target is a candidate for a certain type of targets, also it can say how was it found, is it still on-going and is worth observing, can also indicate the way how to conduct the follow-up observations. Similar details can also be provided in the Comments below the target. \n\n### Phot.Class\nExperimenal classification of the source based on the broad range of archival photometric multi-wavelength measurements from Simbad and other archives. Described in Gezer et al. (2022) https://arxiv.org/abs/2201.12209v2. \nTypically, if the probability is higher than 90%, it can be assumed reliable, otherwise ignore it.\n\n### Last MJD\nModified Julian Date (MJD) of the last observation stored in the database. The observation can be either from one of the follow-up telescopes, or from a survey or archive.\n\n### Last Mag/Filter\nThe most recent magnitude measurement of the last observation. When possible, the measurement is standardised to Gaia-like r or R bands for easier comparision between sources. In case of data from surveys using non-standard filters, e.g. ATLAS, the original filter is reported.\n\n### Target Importance\nThe subjective value of how important is this target. If 0, it means the target is no longer requested for any follow-up observations. If 10 - the target has the highest possible priority in the follow-up requirements. But do check Cadence and Observing Priority.\n\n### Cadence requested\nIf the follow-up is requested (Importance>0), this is the cadence (in days) this target should be observed. By default, it is assumed gri or BVRI filters once per visit are requested. In case of any other requirements, check Description and/or Comments. \n\n### Observing priority\nDynamically computed number. This is a difference between the current MJD and the last MJD, divided by the Cadence requested and the whole thing is multiplied by the Importance. The value is useful for sorting target and is taking into account both how important a target is and when was it last observed, with respect to the requested cadence.\n\n### Sun Separation\nThe angular distance between the target and the current Sun position in decimal degrees. This can serve as a crude estimator of the target visibility and can be used for sorting/filtering. Typically, Sun Separation larger than 60 deg indicates the target should be visible. \n\n<!--  will be added later\n### Target ID\n- Internal numeric/UUID used by the database and the API. -->\n\n\n### Other names/aliases\n- Alternate names pulled from catalogs or added by users (e.g., TNS, ZTF, Gaia IDs).\n- Some names give link to direct webpage of the target on a different service.\n- Note that CRTS, NEOWISE, ATLAS - for these the names are artifically created at the level of BHTOM just based on target's coordinates and should not be used. They are just indicators that the target has potentially some data present in these surveys.\n\n---\n\n## Right column — Tabs\n\n### 1) Photometry\nDisplays all photometric measurements available for the target. It combines both archival and follow-up data. \nThere are two separate ways of displaying the light curves. \n\n#### (a) Photometry by filter\nTop one shows the data sorted by the filter of observation. The follow-up data obtained via BHTOM processing are standardised mostly to Gaia Synthetic Photometry (GaiaSP), and are displayed for example as i(GaiaSP) for i-band (Sloan filter as standardised to GaiaSP). \nThe archival or survey data are not yet standardised and are displayed in their orignal form, for example ZTF(zi) is the i-band Sloan data from ZTF, or GSA(G) is for Gaia Science Alerts G-band. Hovering over a datapoint will show a floating window with the details on the observation - which telescope/user/survey collected the data, its magnitude, error and Julian Date. \n\nThe plotting library is plotly from Python. It allows for interacting with the data. Zoom can obtained by drawing a rectangle on the figure. There is also a possibility to change ranges of Date and Mag axes, by going to the axis and clicking on it when an arrow appears: double-sided arrow (clicked near the centre of the axis) allows for shifting the view; right-sided (top-sided) arrow (near the right/top edge of the axis), changes the upper limit of the range, while the left-sided arrow (down-sided), changes the lower boundary of an axis. \n\nColor-coding tries to represent the filter. B is blue, g/V are typically green, r/R are red, i/I are deep red. Other non-standard bands are shown with different colors. The color palette in this figure is fixed.\n\nLegend can be interacted with as well. Clicking on a single entry, switches its visibility on/off. Double click on an entry hides all but this one. \n\n#### (b) Photometry per facility\nAs for photometry, but this time the legend shows the facilities and telescopes used to collect the photometry. Each follow-up telescope is listed separately and is shown in different color. The color-pallete is random and can vary everytime the page is visited.\n\n> Notes:\n> - Axes: magnitude is inverted (smaller = brighter) and is shown on the left\n> - If radio data is available, the flux axis is added on the right and is not inverted (smaller value=lower flux)\n> - Time base on X-axis is given as a UT Date-Time. \n> - Hovering on a datapoint will show a box displaying the Julian Date (JD), filter this datapoint was standardised to, telescope/camera, observer's name, magnitude/flux and error.\n\n---\n\n### 2) Models\nIn this place we will be adding various astrophysical models for the time-series.\nCurrently, there following models are implemented:\n- Microlensing standard\n- Microlensing with the parallax\n\n<!-- TODO -->\nMore details on how to use these models are not yet provided. \n\n\n### 3) Spectroscopy\n- Interactive spectral viewer a spectrum of the target. \n- This feature will be expanded in the future.\n\n---\n\n### 4) Publication\n- This tab provides target details useful for publication.\n- Table with the statistics on the time-series data stored in BHTOM groups facilities/observers and provides a list of filters used, the number of datapoints and the min-max range of the observations in MJD (Modified Julian Date).\n- The photometric time-series statistics can be downloaded in form of a LaTeX file with the blue button below the table.\n- There is also a table showing Gaia DR2 (GDR2) and Gaia DR3 (GDR3) details on the target found at these coordinates, such as parallax, proper motion in RA and Dec, RUWE (equivallent of the goodness of fit for Gaia astrometry) and the distance estimate based on Gaia data (from Bailer-Jones+2023).\n\n---\n\n### 5) Manage Data\n\n- This page allows user to upload and view various dataproducts to BHTOM.\n- You can upload up to 5 files at once here, if you need to upload more, please check the API Documentation and use the scripts provided there.\n- Supported types:\n  - **Photometry** (SExtractor format) - single catalog file (.dat) containing the SEXtractor header describing the columns. We will only use ALPHA_J2000, DELTA_J2000 and MAG_AUTO and MAGERR_AUTO columns, other will be ignored. You will need to provide the MJD for the time of the observation in the form below.\n  - **Photometry** (CSV) - Lightcurve, time-domain data, please check the example files lined there. Note, the photometry should contain MJD time format. Non-detections can be marked with magnitude error >= 99.0 (e.g. 99.0, 99.9 etc.)\n  - **FITS File** - image in FITS format. The PSF/Aperture photometry will be attempted on this image. You telescope has to be added and registed in BHTOM (see Observatory main menu) and you should have this observatory among your favourite observatories (it should show up in the scroll-down list in the form below).\n  - **Spectra** (ASCII) - please see the example file for the exact format  \n- Parameters:\n  - **Dry Run** default is False and your data will be stored in the DB. Use Dry Run if you want to test if your file processes without errors.\n  - **Observer(s)** By default, the name of the person logged-in is shown. It can be edited, removed, and more people can be added. They have to be registered users of BHTOM, you can search with their login name or full name. Every user's name listed here will be attached to the uploaded dataproduct. This is a good place to add your students taking the observations, or your supervisor, or a person who provided the observing time or the PI of the proposal. These details will be then used in any future publications. \n  - **Observatory/Camera** Please provide the name of the observatory and the camera used for this dataproduct. If your observatory/camera is not on the scroll-down list, you need to add it to your favourite list in the Observatory tab. You will need to refresh the target page for the observatory to show up.\n  - **Comment** Any comment regarding the data, e.g. thin clouds, high humidity, etc.\n- Hit **Upload** to upload your data.\n- The most recent dataproducts uploaded for this target are listed below the Upload button. You can see the details of each dataproduct by clicking Details.
